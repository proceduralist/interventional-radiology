# MASTER SPEC — IR Casebook Platform & Academic IR RPG

**Version 1.0 — 2026-07-02.** Single source of truth for the combined website + game project. Supersedes the two prior game-design drafts; all decisions below are locked with Ryan unless amended here.

---

## 1. Vision

One platform, two faces, one database:

1. **IR Casebook (website)** — the existing public educational reference (51 procedures, cited), extended into a live, Supabase-backed database with an admin dashboard for structured literature entry.
2. **The Academic IR RPG (game)** — a 2D top-down University of Massachusetts Chan Medical School, Worcester, MA-set tactical RPG in which every clinical number is drawn live from that same database.

**The accuracy contract (non-negotiable):** nothing clinically presented as real is invented. Every value is tagged with a data class:

| Class | Meaning | Examples | Display rule |
|---|---|---|---|
| **CITED** | Taken directly from literature/IFUs/guidelines | Device dimensions, complication rates, SIR lab thresholds, contrast/eGFR limits, fluoro reference levels, trial Table 1 baselines | Always shows PMID/DOI/source, on website and in-game codex |
| **MODELED** | Gameplay abstraction, informed by evidence where it exists | 1–10 stiffness/lubricity scales, vessel fragility, BP-decay curves, tactile resistance | Explicitly labeled "simulation parameter" everywhere it appears |
| **DESIGN** | Pure game economy/balance | Device shop prices, payouts, clout tiers, scoring weights | Never presented as clinical fact |

Ryan (physician) is the sole data-entry gate. No automated scraping of JVIR full text (paywall + fabrication risk). The dashboard may prefill paper *metadata* (title/authors/year/abstract) from a pasted PMID/DOI via the free PubMed/Crossref APIs; all structured numbers (N, endpoints, p-values, AEs, baselines) are entered by hand.

## 3. Architecture

**Everything stays buildless and statically hosted; Supabase provides the entire backend.**

```
Repo layout (this folder, deployed as one GitHub Pages site):
/                      → IR Casebook (existing index.html, app.js, styles.css, sw.js)
/admin/                → Data-entry dashboard (vanilla JS + supabase-js via CDN; admin login)
/game/                 → Phaser 3 game (CDN Phaser + plain JS modules + Tiled JSON maps)
/game/assets/          → CC0 tilesets, sprites, audio (licenses recorded in CREDITS.md)
/db/                   → schema.sql, seed + migration scripts, snapshot exporter
/docs/                 → this spec and design docs
```

- **Supabase (free tier):** Postgres + Auth + auto-generated REST (PostgREST). The `anon` key is public by design; Row-Level Security enforces permissions (public read on content, admin-only write, owner-only saves).
- **Casebook offline mode is preserved:** the site reads Supabase when online and falls back to a service-worker-cached JSON snapshot (`procedures.json` becomes a generated snapshot artifact, no longer the master).
- **Website→game sync:** every content write bumps `data_version` (DB trigger). The game checks the version at boot (and on returning to the hospital hub) and refetches changed tables. No deploys needed for content changes.
- **No bundler anywhere.** If `/game` complexity ever demands it, introduce Vite for that folder only.

### Migration of existing content (P0)
`db/migrate_procedures_json.py` parses the current `procedures.json` (51 procedures, 10 categories) into `categories`, `procedures`, `citations`, and `complications` rows. `validate.py` logic becomes DB constraint checks. The casebook UI is refactored to read the snapshot/DB with zero visual change.

## 4. Database (see `db/schema.sql` for full DDL)

**Content (public read / admin write):** `categories`, `procedures` (reference content, jsonb sections matching today's schema), `procedure_game_params` (vessel map ref, access/target nodes, fluoro target + citation, payout, pacing, complexity), `citations`, `papers` (structured trials: N, design, inclusion/exclusion, endpoints w/ p-values & CIs, adverse events, Table 1 baselines), `procedure_papers` (junction), `devices` (dimensions CITED, gameplay scales MODELED), `vessel_maps` (node/edge graphs), `complications` (universal + specific, rates, severity per SIR AE classification, rescue actions), `patient_generators` (sampling distributions sourced from papers), `npc_archetypes`, `defense_question_templates`, `game_config` (tunable constants with data-class tags).

**Player (owner-only via RLS):** `profiles` (with `is_admin` flag for Ryan), `player_saves` (3 slots, jsonb state), `case_logs` (per-case score breakdowns feeding achievements).

**Game-readiness rule:** a procedure auto-appears in the game when the `v_game_ready` view finds it published with game params, a vessel map, a patient generator, and ≥1 complication — "all procedures on the website are in the game" stays true by construction.

## 5. Data pipeline (Ryan's workflow)

Dashboard workflows at `/admin`:

1. **Procedure editor** — same fields as today's JSON schema, with inline citation manager.
2. **Paper entry** — paste PMID/DOI → metadata prefills (PubMed/Crossref, free APIs) → Ryan enters structured fields into validated forms: cohort sizes, inclusion/exclusion, endpoints (effect size, CI, p), adverse events, Table 1 rows (variable, mean±SD or n/%, by arm).
3. **Device entry** — dimensions/coatings from IFUs (CITED, source URL required) + MODELED 1–10 gameplay scales with a required "basis" note (e.g., "hydrophilic coating per IFU → lubricity 9").
4. **Complication & rescue editor** — rate text + parsed numeric range, severity (SIR AE class), management text, structured rescue actions.
5. **Vessel map editor** — v1: JSON textarea + schema validation and live graph preview; visual node editor later.
6. **Patient generator builder** — pick a linked paper, transcribe Table 1 rows into distributions; set guardrails.

Publishing anything bumps `data_version` → live in game on next boot.

## 6. Patient generation (grounded RNG)

Each case's patient is sampled from `patient_generators`:

- **Distributions (CITED):** age, sex split, weight/BMI, creatinine/eGFR, Hgb/platelets/INR, comorbidity prevalences (CKD, DM, HTN, cirrhosis...), anticoagulant use — taken from the baseline table of a real cohort for that indication, citation attached.
- **Guardrails (clinical logic):** sampled values are mutually constrained — e.g., Cr 2.4 ⇒ CKD tag ⇒ eGFR computed (CKD-EPI 2021, already a site calculator) ⇒ contrast limit = min(Cigarroa V_max = 5 mL×kg/SCr, ACR eGFR-tier limit); INR/platelets vs SIR 2019 periprocedural thresholds (already on the site) gate whether the case may proceed or needs correction orders; anticoagulants imply hold/bridge decisions in pre-op.
- **Chart realism:** generated EMR shows HPI, PMH, meds, labs with flags, and prior imaging notes. Every number traceable to its generator source.

## 7. The Angio Suite engine

### Telemetry (per-turn hidden physiological engine)
SBP / DBP / HR / SpO₂ / RR (sedation-sensitive) as live vitals; accumulators for **fluoro time** and **reference air kerma (mGy)** and **contrast (mL)**; **ACT** when heparinized; **Tactile Resistance** readout (LOW / MODERATE / CRITICAL) for the current tool-in-vessel. Hemorrhage response follows a MODELED compensatory-shock curve (informed by ATLS shock classes: tachycardia before hypotension; DBP falls slower). SBP ≤ 60 ⇒ code (critical failure). Sedation meds depress RR/SpO₂; reversal agents exist.

### Pacing
- **Stable/elective:** pure turn-based (ports, UFE...).
- **Emergent:** real-time degradation between actions (GI bleed, trauma) — MODELED decay rates per case.

### Action taxonomy (v1 set; every action logs to the score ledger)
Access (US-guided micropuncture), wire advance/pullback/exchange, catheter/microcatheter advance/exchange, sheath up/down-size, imaging (fluoro tap, DSA run, roadmap — each adds dose; collimation/mag/frame-rate choices modify dose rate), contrast injection (volume chosen per puff/run), therapy (balloon inflate, stent deploy, coil deploy, particle aliquots, lytic drip), meds (heparin+ACT, atropine, sedation titration, reversal, fluids, pressors), closure (manual pressure vs device), and case-specific steps (pocket creation, tunneling, catheter trim for ports).

### Vessel stress & complications
`Vessel Stress = (Tool Stiffness × Node Tortuosity / Tool Lubricity) × Push Force` (MODELED). Stress above a node's fragility triggers a complication roll weighted by that complication's **CITED** incidence rate. Trigger ⇒ **Emergency Mode**: navigation locks, vitals decay, and the player must select a valid rescue action (from the complication's DB row) before decompensation.

### Scoring (0–100, hidden until post-op debrief)
- **Patient Safety (40):** area-under-SBP-100 across turns + consecutive shock turns.
- **Radiation (20):** vs the procedure's CITED fluoro-time/dose reference level.
- **Renal Protection (20):** vs the patient's computed contrast limit (§6).
- **Technical Skill (20):** CRITICAL-resistance pushes, wasted devices, guideline violations (e.g., proceeding past lab thresholds without correction).
Debrief screen itemizes every deduction against its log entry — the scoring ledger doubles as teaching.

## 8. Devices (zero-sum tiers)

Attributes: dimensions/French/lengths/coating/tip shape (**CITED**, IFU-sourced) + stiffness, lubricity, trackability, support, visibility 1–10 (**MODELED**, basis note required) + tier and price (**DESIGN** — real list prices are rarely public; if Ryan enters a published cost analysis, the admin NPC and shop may cite it). Tier ladder as previously drafted (Bentson/C2 stock → Glidewire/Glidecath → Progreat/Renegade micro-systems), balanced zero-sum: no dominant tool. Compatibility rules (wire OD vs catheter ID, sheath French) enforced by the engine.

## 9. Simulation Lab

- **Flow Bench:** pick synthetic anatomy profile + any owned wire/catheter combo → live readouts of friction/support/perforation risk computed from the same MODELED attributes the real engine uses (so practice transfers).
- **Spec Bench:** device detail view — CITED dimensions with IFU links, tip architecture, shaping tips.
- **Embolic Playground:** coil packing-density sandbox (packing % = coil volume / target volume — published formula, cited) and particle-size vs occlusion-level demos.

## 10. Conference defense

Green-room dossier = the actual `papers` row. Podium gauntlet spawns 3–5 critics from the 7 archetypes (biostatistician, old-school surgeon, panicked fellow, cost-conscious admin, rival author, lost med student, industry consultant). Questions are template-generated against paper fields; **correct answers are always the literal entered data**; distractors are systematic perturbations (wrong N, inverted CI, violated exclusion criterion). Credibility gauge 100→0; wrong answers −20–30; below 30 ⇒ ejected, zero rewards, reputation penalty. Hand-authored questions can be added per paper in the dashboard.

## 11. Overworld, economy, saves

- **University of Massachusetts Chan Medical School, Worcester, MA (stylized, 16-bit):** the University Campus site plan (May 2025) rendered as a scrolling 80×100-tile world (32px tiles, camera follows the player) — Plantation St/Lake Ave/Belmont St street grid, Lake Quinsigamond, biotech park, quads, garages, helipad, and 21 campus buildings shown in 3/4 top-down (roof + south wall face). **Campus map = fast travel** (press M, click a building to ride the shuttle; walk anywhere scenic). Enterable buildings open interiors: UMass Memorial → 3rd-floor IR hub (inpatient ward EMR at bedside, angio suite, sim lab, procurement; call room/lounge planned); all others → lobby scenes with flavor + signposts for future arcs (Sherman auditorium → P3 conference, Lazare → research, ACC → outpatient clinic).

Clout tiers: 0–50 Resident (baseline) / 51–150 Rising Researcher (1.2× payouts, faster procurement) / 151–300 KOL (matching grant, once per cycle) / 301+ Academic Chief (1.5×, instant supply chain). All DESIGN values in `game_config`.
- **Saves:** Supabase Auth (email/password) from day one; 3 save slots; autosave on zone transitions and case completion; owner-only RLS.
